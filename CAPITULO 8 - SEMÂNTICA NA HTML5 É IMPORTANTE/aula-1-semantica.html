Semântica na HTML5 é importante

Algumas tags em HTML possuem um significado aos comandos incluídos no desenvolvimento.

A tag address é utilizada para mencionar um endereço.

A tag strong possui o valor de negrito, pois dá força ao comando digitado


Alem de strong, possuem função semântica na estrutura da página que será desenvolvida:


A tag article, utilizada para conteúdos independentes e autocontidos. Exemplo: Postagens de fórum, postagens de blog, comentários do usuário, cartões de produtos e artigos de jornal.

A tag aside, ou "à parte", define algum conteúdo além do conteúdo em que é colocado (como uma barra lateral). Seria um tipo de "referência", ou seja, explicar algo que é mencionado em outra parte e/ou tag.

A tag details define detalhes adicionais que o usuário pode visualizar ou ocultar.



A tag footer, ou rodapé, é utilizada, como o próprio nome sugere, como rodapé de um documento ou seção. Exemplos: informação de autoria, informações sobre direitos autorais, informação de contato, mapa do site, voltar ao topo links, documentos relacionados.

A tag header, utilizada como um contêiner para conteúdo introdutório ou conjunto de links de navegação. Exemplo: um ou mais elementos de título (H1 - H6), logotipo ou ícone, informações de autoria.

A tag main especifica o conteúdo principal de um documento

A tag nav define um conjunto de links de navegação.

A tag section, utilizada para definir uma seção de documento (agrupamento temático de conteúdo, normalmente com um título). Exemplos: Capítulos, Introdução, Novos itens, Informações de contato.

A tag summary define um cabeçalho visível para um elemento details
