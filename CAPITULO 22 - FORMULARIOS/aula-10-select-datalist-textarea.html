Exemplo de Formulário


SELECT

O recurso SELECT é utilizado para caixas de diálogos em que o usuário, ao clicar na seta posicionada no lado direito deste abre uma lista de opções.

OPTION

O recurso OPTION é justamente a opção que será incluída no recurso SELECT.

OPTGROUP

Para que possamos criar subclasses dentro do recurso SELECT (como demonstrado no presente exercício - Região Sudeste e Região Nordeste), envelopamos todos os options que são da mesma subclasse com o recurso OPTGROUP.

DATALIST - DATAL

O recurso DATALIST também cria uma lista de opções para que o usuário, ao selecionar a caixa de diálogo, as escolha.

Contudo, há uma diferença entre DATALIST e SELECT. Enquanto o último já cria uma caixa de diálogo com a seta de seleção, o DATALIST precisa de uma vinculação entre as options e o input text.

Primeiro cria-se um id para o DATALIST e depois insere esse id no recurso LIST, inserido no item input

MAS ENTÃO, POR QUE EU USARIA O DATALIST?

Simples, pois o DATALIST permite que o usuário inclua um item que não existe na lista pré-elaborada. Exemplo: Se eu for cozinheiro, apesar da profissão não estar na lista pré-elaborada, eu consigo inserí-la como opção.

TEXTAREA

O TEXTAREA é o recurso utilizado para se criar uma caixa de diálogo em que o usuário poderá escrever um texto (Geralmente, é utilizado para opções de reclamações, suporte, etc, em que o usuário precisa explicar quais são suas necessidades).

Para a inclusão desse recurso, é importante manter o method POST do formulário, tendo em vista a limitação de 3 mil bytes do method GET.
