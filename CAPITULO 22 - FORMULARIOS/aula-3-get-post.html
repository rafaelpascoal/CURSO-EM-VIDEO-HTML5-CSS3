GET E POST

MÉTODOS DE SEGURANÇA EM ENVIO DE FORMULÁRIOS: GET E POST

Por padrão, se não há inclusão de método (method), o recurso que será habilitado é o GET (situação que não fornece segurança aos dados inseridos pelo usuário, pois as informações aparecem na url).

Podemos utilizar o método GET nas seguintes situações (e quando os dados a serem inseridos não forem sensíveis):

  • Nome;
  • Peso;
  • Altura;
  • Cor de olhos;
  • Em links que a intenção é que sejam comparilháveis;
  • Dados de até 3000 bytes;
  • ETC

Com o método POST, as informações inseridas pelo usuário não aparecerão na url, mas isso não quer dizer que e o método mais seguro de todos, pois existem formas de acessarmos os dados inseridos pelo usuário. (clicar com o botão direito na tela do formulário e selecionar inspecionar -> Network -> Sececionar o arquivo de envio dos dados -> Headers)

Já o método post deve ser utilizado nas seguintes situações (e quando as informações forem sensíveis):

  • Senhas;
  • Dados Bancários;
  • Dados fiscais;
  • Em links que a intenção é que não sejam comparilháveis;
  • Número de cartão de crédito.

Portanto, é necessário utilizar o HTTPS que é mais seguro (outro patamar de estudos).
